Synthèse

Contentstack est un CMS headless et composable : il stocke et livre le contenu via API, indépendamment de tout frontend. Architecture MACH (Microservices, API-first, Cloud-native, Headless).

Hiérarchie des entités

NiveauEntitéAnalogieRôle
1OrganisationL'entreprise (l'étagère)Conteneur global, gestion des utilisateurs et stacks
2StackUn projet / site (le livre)Repository de tout le contenu d'un site

Types de Content Types

Single
Page unique

Une seule entry possible. Ex : Homepage, À propos, CGV.

Multiple
Pages multiples

N entries du même schema. Ex : fiches produit, articles de blog.

Multiple + Référence
Composant réutilisable

Entries non-page référencées ailleurs. Ex : auteurs, catégories.

Field Visibility Rules

Afficher / masquer des champs selon la valeur d'un autre champ. Max 10 règles par CT, 5 conditions par règle, 5 champs cibles par règle. Réservé aux rôles Owner / Admin / Developer.

Publishing

ConceptDéfinition
EnvironmentServeur de destination (Dev / Staging / Prod). Contenu publié = accessible via API sur cet env.
Live PreviewPreview en temps réel sans publier. Nécessite une config développeur. HTTPS obligatoire.
Publish QueueFile d'attente des actions publish/unpublish. Icône verte = dernière version · orange = version ancienne · gris horloge = schedulé.
ReleasesLot d'entries/assets à publier simultanément. Max 500 items. Idéal pour lancements ou promos.
SchedulingPublication différée : sélectionner "Later" + date/heure lors du publish.
Localization+200 locales disponibles. Langue de fallback configurable. Héritage sur 1 niveau.
VersioningChaque save = nouvelle version numérotée. Versionnage entries ET assets. Nommage personnalisable.

Mapping métier — Cas d'usage Etam

Besoin EtamConcept ContentstackConfig recommandée
Fiche produit Content Type "Multiple" Fields : Title, Description (HTML RTE ou Multi-line), File (image produit), Select (catégorie, couleur, taille), Number (prix), Boolean (actif/inactif), Reference (catégorie, marque)
Homepage saisonnière Content Type "Single" + Modular Blocks Blocs : Hero banner, Carrousel promo, Push catégorie, Bandeau transport — chaque bloc = un composant indépendant, ordonnancement libre sans dev
Page campagne / collab (ex: Etam x Rouje) Content Type "Single" ou entrée dédiée URL field custom (/collab-2), Modular Blocks pour structurer les sections, scheduling pour le teaser
Gestion des catégories Content Type "Multiple" réutilisable Entries catégories référencées dans les fiches produit via Reference field · mise à jour en un point, propagée partout
Contenu multilingue (FR/BE/ES…) Localization Master = fr-FR · Fallback par marché · localiser uniquement les champs texte, garder assets non-localisés (Non-localizable)
Lancement promo / soldes Releases + Scheduling Préparer toutes les entries dans une Release · scheduler le deploy à J-1 minuit · unpublish schedulé en fin de période
Bandeau transport / port offert Global Field ou Content Type Single Global Field "Bandeau alerte" (texte + lien) réutilisé sur toutes les pages · Boolean pour activation rapide
Enrichissement descriptions produit HTML RTE ou JSON RTE JSON RTE recommandé : sortie JSON consommable par le moteur de rendu, compatible multi-device · possibilité d'embed assets inline
SEO pages catégories / LP Global Field "SEO" Champs : SEO Title (Single line), Meta Description (Multi-line), OG Image (File) · Global Field réutilisé sur chaque CT de page
Workflow de validation contenu Workflows + Publish Rules Stages : Rédaction → Relecture → Validation Merch → Publish · Publish Rule = blocage publish Prod sans passage au stage "Validé"
Gestion équipe contenu Roles personnalisés Rédactrice : Write entries · Merch Lead : Approve + Publish · Dev : accès Content Type · interdire publish Prod aux rôles non valideurs
Preview avant mise en ligne Live Preview Nécessite config dev côté frontend · permet de voir les modifs en temps réel sans sauvegarder · à configurer par env (staging)
Labels / organisation des CT Labels Ex : label "Campagnes", "Produits", "SEO", "Transport" · visibles de tous les users du stack · hiérarchisables

Workflow éditorial recommandé Etam

Rôles à configurer

Rédactrice contenu

  • Create / Edit entries
  • Upload assets
  • Accès staging preview
  • Pas de publish Prod

Merch / Responsable

  • Review entries
  • Valider workflow stage
  • Publish staging
  • Approbation publish Prod

Développeur

  • Créer / modifier Content Types
  • Config environments
  • Config Live Preview
  • Config Field Visibility Rules

Admin Stack

  • Gérer utilisateurs
  • Créer workflows
  • Définir Publish Rules
  • Toutes permissions

Stages workflow recommandés

Brouillon
→
En relecture
→
Validé Merch
→
Prêt à publier
→
Publié Prod

Publish Rule : publish sur Prod bloqué tant que le stage "Prêt à publier" n'est pas atteint + approbation Merch Lead requise.

Statuts publish queue

Vert = version actuelle publiée Orange = version ancienne publiée Gris horloge = publication schedulée Pas d'icône = jamais publié
